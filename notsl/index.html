Transformational Technologies

Data transformation has technological components too. Just as we can transform information into a myriad of formats, we have variety of means to perform the transformation.
A card catalog functioned as the data format, storage, search, index and presentation. An OPAC has different software components that function for each of these things. MARC is the standard, but there are "stacks" of technology that ultimately get the information to the user's eyes.

Tech Components

  • presentation: html
  • searching: Blacklight
  • indexing and storage: Solr
  • Ruby, Rails
These are the components we chose to transform our data. The data format, indexing and presentation are all separate.

Transforming EAD

Tranforming MARC is easy, fundamental use for Blacklight. How was I going to use these tools to transform our massive finding aids?
The solution was to quite literally, blow them up into lots of tiny pieces

Breaking up is hard to do...

  • one piece for the finding aid
  • one piece each for every series, sub-series, folder, etc.
  • one piece each for every item
  • thousands of pieces per collection

Searching and Relevance

  • MARC-based items interfiled with items from collections
  • boosts the relevance of items, not just collections
  • exposes items across and within collections
  • faceted search capabilities
  • minimal Schema.org integration
  • container-level pieces discarded from search results

Presentation

  • pieces are re-assembled for finding aid display
  • only relevant sections are displayed
  • default view does not show the entire finding aid
  • because people don't read web pages [1]
